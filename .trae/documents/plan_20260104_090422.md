# 编写与Godot一致的ResourceUID测试用例

## 1. 创建测试项目
- 在当前解决方案中添加一个新的xUnit测试项目
- 命名为GodotResourceUID.Tests
- 引用主项目GodotResourceUID

## 2. 编写测试用例
根据Godot的test_resource_uid.h文件，编写三个测试用例：

### 测试用例1：最大/最小UID的编码/解码
- 测试最大UID (0x7fffffffffffffff) 的编码结果是否为 "uid://d4n4ub6itg400"
- 测试 "uid://d4n4ub6itg400" 的解码结果是否为 0x7fffffffffffffff
- 测试最小UID (0) 的编码结果是否为 "uid://a"
- 测试 "uid://a" 的解码结果是否为 0

### 测试用例2：无效UID的编码/解码
- 测试无效UID (-1) 的编码结果是否为 "uid://<invalid>"
- 测试 "uid://<invalid>" 的解码结果是否为 -1
- 测试无效UID (-2) 的编码结果是否与-1相同
- 测试没有scheme的UID "dm3rdgs30kfci" 的解码结果是否为 -1

### 测试用例3：各种UID的编码/解码
- 测试UID 1 的编码结果是否为 "uid://b"
- 测试 "uid://b" 的解码结果是否为 1
- 测试UID 8060368642360689600 的编码结果是否为 "uid://dm3rdgs30kfci"
- 测试 "uid://dm3rdgs30kfci" 的解码结果是否为 8060368642360689600

## 3. 运行测试
- 使用dotnet test命令运行测试
- 确保所有测试用例都通过

## 4. 验证一致性
- 所有测试用例的预期结果都与Godot的测试用例完全一致
- 测试用例的结构和逻辑也与Godot的测试用例保持一致
